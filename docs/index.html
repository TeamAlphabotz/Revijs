Four steps to full SEO

ReviJs plugs into your existing build process. No rewrites. No new frameworks.

01
🏗️
Build your app
Run your normal npm run build. ReviJs works with your existing dist/ output.
02
🚀
Run ReviJs
Run npx revijs. It spins up a local server and a headless browser automatically.
03
📸
Capture HTML
The browser visits each route, waits for data, then captures the full rendered DOM.
04
🤖
Serve to bots
Bots get static HTML. Real users still get your smooth SPA. Best of both worlds.

Everything you need

Designed to be dropped into any existing React or Vite project in seconds.

🖥️
Headless Browser Engine
Powered by Playwright Chromium internally. Fully abstracted — you never install or configure it manually.
🔒
Local-First & Private
Everything runs on your machine. Your code, data, and API responses never leave your server.
🤖
Bot Detection Middleware
Drop-in Express middleware detects 30+ crawlers and serves them prerendered HTML automatically.
⚙️
3 Rendering Engines
Choose from browser, advanced, or ssr mode depending on your app's needs.
📦
Zero Config to Start
Run npx revijs init to generate a starter config. One command, you're ready.
🔌
Programmatic API
Import and call prerender() from your own scripts for full programmatic control.

// config

Simple config file

Drop a revi.config.js in your project root. That's all ReviJs needs.

revi.config.js
export default {
  // Routes to prerender
  routes: ['/', '/about', '/blog/post-1'],

  // 'browser' | 'advanced' | 'ssr'
  engine: 'browser',

  outputDir: 'dist-prerendered',
  distDir: 'dist',

  // ms to wait after network idle
  waitFor: 1200,
  headless: true,
  port: 4173,
};
