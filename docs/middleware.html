Middleware

The optional ReviJs middleware intercepts bot requests on your server and serves prerendered HTML, while real users continue to get your normal React SPA.

How it works

🌐
Request comes in

Any visitor requests a page from your server.

🔍
Middleware checks User-Agent

ReviJs checks the User-Agent header against 30+ known bot patterns (Googlebot, GPTBot, ClaudeBot, etc.).

🤖
Bot → prerendered HTML

Bots get the static HTML from dist-prerendered/. Full content, no JavaScript needed. Perfect for SEO.

👤
Human → normal SPA

Real users pass through to your React app as normal. Full SPA experience unchanged.

Express

server.js
import express from 'express';
import { createMiddleware } from '@revijs/core';

const app = express();

// ReviJs intercepts bots before anything else
app.use(createMiddleware({
  prerenderedDir: 'dist-prerendered',
  debug: false,
}));

// Normal users get the SPA
app.use(express.static('dist'));

app.listen(3000);

Polka (lightweight alternative)

server.js
import polka from 'polka';
import sirv from 'sirv';
import { createMiddleware } from '@revijs/core';

polka()
  .use(createMiddleware({ prerenderedDir: 'dist-prerendered' }))
  .use(sirv('dist', { single: true }))
  .listen(3000);
💡 Optional

The middleware is completely optional. If you deploy to a CDN or static host that serves different files per User-Agent (e.g. Cloudflare Workers, Vercel Edge), you don't need this at all.
