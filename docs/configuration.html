Configuration

ReviJs is configured via a revi.config.js file in your project root. Run npx revijs init to generate one automatically.

Full Config Reference

revi.config.js
export default {
  routes: ['/', '/about', '/blog/post-1'],
  engine: 'browser',
  outputDir: 'dist-prerendered',
  distDir: 'dist',
  waitFor: 1200,
  headless: true,
  port: 4173,
};

Options

OptionTypeDefaultDescription
routesstring[][ ]Array of route paths to prerender. Required.
enginestring'browser'Rendering engine to use. See engines section below.
outputDirstring'dist-prerendered'Directory where static HTML files are written.
distDirstring'dist'Your built SPA directory. Run npm run build first.
waitFornumber1200Milliseconds to wait after network idle before capturing HTML.
headlessbooleantrueRun browser in headless mode. Set false to watch the browser.
portnumber4173Port for the temporary local server. Auto-increments if taken.
debugbooleanfalseEnable verbose logging. Also available as --debug CLI flag.

Rendering Engines

Choose an engine based on your app's complexity:

'browser' ← default

Standard headless Chromium. Works for 99% of React/Vite apps. Navigates, waits for network idle, captures DOM.

'advanced'

Like browser, but also simulates scroll for lazy-loaded content and injects a x-prerendered-by meta tag.

'ssr'

Minimal-wait mode for apps with true server-side rendering. Skips the waitFor delay.

💡 Tip

Not sure which engine? Start with 'browser'. Only switch to 'advanced' if you have lazy-loaded components that aren't appearing in the output.
