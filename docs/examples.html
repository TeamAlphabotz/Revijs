Examples

Real-world config examples for common app types. Copy and adjust for your project.

Blog / Content Siteblog

A typical blog with a listing page and individual post pages. Increase waitFor if your markdown renders slowly.

revi.config.js
export default {
  routes: [
    '/',
    '/blog',
    '/blog/hello-world',
    '/blog/getting-started-with-react',
    '/about',
    '/contact',
  ],
  engine: 'browser',
  waitFor: 2000,
  outputDir: 'dist-prerendered',
};

E-Commerceecommerce

Prerender your product pages and landing pages. Skip auth-protected routes like /checkout and /account.

revi.config.js
export default {
  routes: [
    '/',
    '/products',
    '/products/sneaker-x1',
    '/products/sneaker-x2',
    '/categories/shoes',
    '/about',
  ],
  // Skip: /cart /checkout /account
  engine: 'browser',
  waitFor: 1500,
};

SaaS Landing Pagesaas

Only prerender your public-facing marketing pages. The app dashboard stays as a normal SPA.

revi.config.js
export default {
  routes: [
    '/',
    '/pricing',
    '/features',
    '/changelog',
  ],
  // /app/* and /dashboard/* are skipped (auth-protected)
  engine: 'browser',
  waitFor: 1200,
};

CI/CD Build Scriptci

Automate prerendering in your build pipeline by adding a prerender script to package.json.

package.json
{
  "scripts": {
    "build": "vite build",
    "prerender": "npm run build && npx revijs",
    "deploy": "npm run prerender && netlify deploy --dir=dist-prerendered"
  }
}
